JavaBean 类

要求

  1. 类名见名知意
  2. 成员变量 private 修饰
  3. 无参和全参的构造方法
  4. 成员方法

IDEA 使用

javabean

IDEA IDE 快捷键

ctrl+D : 向下复制一行
alt+[fn]+insert :

  • constructor : 构造方法
  • getter and setter : 生成 getxxx() 和 setxxx() 方法

PTG 插件

右键点击 -> Ptg To JavaBean

整理格式

IDEA IDE 快捷键：ctrl+alt+[shift]+L

对选中代码块外包 for 等结构

IDEA IDE: ctrl+alt+T

搜索与跳转

搜索

ctrl+N : 在当前 project 或 all places 或其他位置搜索文件

跳转

ctrl+B 或 ctrl+左键 : 转到定义

继承

继承内容

非私有 private
构造方法 N N
成员变量 Y Y
成员方法 Y N

访问特点

成员变量、方法：就近原则。

构造方法：

  1. 父类构造方法不会被子类继承
  2. 子类中所有构造方法默认先访问父类中的无参构造，再执行自己（子类初始化前，一定要调用父类构造方法先完成父类数据空间的初始化）
  3. 子类构造默认第一行存在 super() 。若想调用父类有参构造，必须手动写 super 调用

方法的重写

  1. 名称、形参列表一致
  2. 子类方法访问权限必须大于等于父类
  3. 子类方法返回值类型必须小于等于父类
  4. 添加到虚方法表中的方法才能被重写

多态

调用成员的特点

变量调用：编译看左边，运行也看左边

方法调用：编译看左边，运行看右边

抽象类与抽象方法

抽象类：如果一个类中存在抽象方法，那么该类就必须声明为抽象类

注意事项

  1. 抽象类不能实例化
  2. 抽象类中不一定有抽象方法，有抽象方法的类一定是抽象类
  3. 可以有构造方法
  4. 抽象类的子类：要么重写抽象类中所有抽象方法；要么是抽象类

接口

  1. 接口不能实例化
  2. 接口的子类（实现类）：要么重写接口中所有抽象方法；要么是抽象类

成员特点

成员变量：只能是常量；默认修饰符： public static final

构造方法：无

成员方法

默认修饰符：public abstract

JDK7 前：接口中只能定义抽象方法

JDK8：接口中可以定义有方法体的方法（默认、静态）

(1) 允许在接口中定义默认方法，使用关键字 default 修饰

格式：public default 返回值类型 方法名(参数列表){}

注意事项：

  1. 默认方法不是抽象方法，不强制被重写。若要被重写，重写时去掉 default 关键字
  2. public 可以省略，default 不能省略
  3. 若实现了多个接口，多个接口中存在相同名字的默认方法，子类就必须对该方法进行重写

(2) 静态方法，使用 static 修饰

格式：public static 返回值类型 方法名(参数列表){}

注意事项：

  1. 静态方法只能通过接口名调用，不能通过实现类名或对象名调用
  2. public 可以省略，static 不能省略

JDK9：接口中可以定义私有方法（普通、静态）

格式1：private 返回值类型 方法名(参数列表){}

格式2：private static 返回值类型 方法名(参数列表){}

内部类

访问特点

  1. 内部类可以直接访问外部类的成员，包括私有
  2. 外部类要访问内部类的成员，必须创建对象

类别

成员内部类

  1. 可被一些修饰符修饰，如: private, 默认, protected, public, static 等
  2. 成语内部类里面， JDK16 开始可以定义静态变量

静态内部类

静态内部类只能访问外部类中的静态变量和静态方法，若想要访问非静态的需要创建对象。

局部内部类

  1. 将内部类定义在方法里面就叫做局部内部类
  2. 外界无法直接使用，需要在方法内部创建对象并使用
  3. 该类可以直接访问外部类的成员，也可以访问方法内的局部变量

匿名内部类

匿名内部类本质就是隐藏了名字的内部类

可以写在成员位置，也可以写在局部位置

格式：

体现了(1)实现/继承(2)方法的重写(3)创建对象

事实上，java会自动给这种类起名字
